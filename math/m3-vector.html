ベクトル ― 位置と向きを「1本の矢印」で持つ

本編で出てきた頂点(x, y, z)も、法線（表面の向き）も、正体は同じ「ベクトル」でした。数の3つ組を1個の矢印として扱う、それだけの話です。

⏪ 本編との接続

  • 本編第1章bの頂点は「(x, y, z)の3つ組」だった
  • 本編第3章aの法線は「表面の向きを表す矢印」だった
  • この2つを1つの言葉で扱えるようにするのがこのページ

1. ベクトル＝「まとめて1個」として扱う数の組

v = (x, y) ベクトル・ブイ。複数の数の組を「1個のもの」として扱うときの呼び名と記法。3Dなら(x, y, z)の3つ組。書物では太字のvや、vの上に矢印を載せた記法を使うが、意味はどれも同じ「数の組に付けた1つの名前」。

座標(2, 1)は「場所」を表しました。同じ(2, 1)を、「原点から右へ2・上へ1だけ進む矢印」と見なすこともできます。この「矢印としての見方」がベクトルです。数字は同じ、解釈が2通りあるだけ。

  • 位置として読む → 頂点の座標（本編第1章）
  • 向き＋長さとして読む → 法線・光の方向・移動量（本編第3章）

矢印として読むとき、大事な情報は2つに分解できます。向きと長さです。下のウィジェットで矢印の先端をドラッグして、成分(x, y)・向き・長さがどう連動するか見てください。

長さ = √( x² ＋ y² )

矢印の長さは、横成分の2乗と縦成分の2乗を足して、平方根（ルート）を取ったもの。中学の三平方の定理そのままです。

2. ベクトルの足し算＝「矢印のしりとり」

ベクトル同士は足せます。規則は拍子抜けするほど単純で、成分ごとに足すだけ。

a ＋ b = ( aₓ＋bₓ , aᵧ＋bᵧ )

2本の矢印の和は、横成分同士の和と、縦成分同士の和を並べたもの。（aₓは「ベクトルaの横成分」という意味の添え字です）

図で見ると、この計算は「aの矢印の先から、続けてbの矢印を歩く」ことに対応します。矢印のしりとりです。「東へ2・北へ1歩いてから、東へ1・北へ2歩く」＝「合計で東へ3・北へ3歩いた」。当たり前の話が、成分の足し算と完全に一致します。下のウィジェットで2本の矢印をそれぞれドラッグして確かめてください。

3. 長さ1の矢印 ― 法線がいつも「単位ベクトル」な理由

本編第3章aで見た法線（表面の向きの矢印）には、業界の決まりがあります。長さを必ず1にそろえるのです。長さ1のベクトルを単位ベクトルと呼びます。

理由は明快で、法線の仕事は「向きを伝えること」だけだから。長さに情報を持たせないと決めておけば、後の計算（次ページの内積）で「向きの違い」だけを純粋に取り出せます。長さをそろえる操作（正規化）は、各成分を長さで割るだけです。

単位ベクトル = ( x長さ , y長さ )

元のベクトルの各成分を、そのベクトル自身の長さで割ると、向きが同じで長さ1の矢印になる。

下のウィジェットで矢印をどれだけ伸ばしても、正規化後の緑の矢印は単位円の上から動きません。「向きだけが残る」を目で確認してください。

WIDGET 3

正規化 ― どんな矢印も長さ1に

「ベクトルは物理や数学の専門概念で、3DCGでは裏方の話」――裏方どころか主役です。頂点も法線も光の向きもカメラの向きも移動量も、GPUが毎フレーム扱うデータのほぼすべてがベクトル。本編で触ってきたデモの中身は、ベクトルの足し算と掛け算の洪水でした。

ベクトルは「数の組を1本の矢印として読む」見方。足し算はしりとり、正規化は向きだけを残す操作。
