負の数と座標平面 ― つまずきの原点を、最初にほぐす

「マイナス×マイナスがなぜプラス？」で数学と気まずくなった人は多いはず。3DCGに必要な範囲だけ、実物を触って仲直りします。

⏪ 本編との接続

  • 本編第1章で、座標＝原点基準の住所、マイナス＝向きが逆なだけ、と学んだ
  • このページはその「マイナス」を、計算まで含めて使える道具にする

1. マイナスの数は「向きを持った数」

普通の数（正の数）は「量」だけを表します。3は3個の3。ところが座標の世界の数は、量と向きの両方を運びます。+3は「プラス方向へ3」、−3は「マイナス方向へ3」。符号（+/−）は向き、絶対値（数字部分）は量。この分解が今日の一番の道具です。

下の数直線で点をドラッグしてください。0の右では符号がプラス、左に越えた瞬間マイナスに変わります。数字の大きさ（量）と符号（向き）が別々の情報として表示されるのを確認してください。

数直線 ― 符号は向き、絶対値は量

2. 「マイナスを足す」＝逆向きに歩く

計算に入ります。まず足し算。「＋」は「その向きに歩く」と読み替えると、負の数の足し算は一切暗記が要らなくなります。

x′ エックスプライムと読む。「変換後のx」という意味の記号。元のxと区別するために右肩に印（′）を付けただけで、中身はただの数。この記号はこの先の全ページで使う。
x′ = x + d

移動後の位置 x′ は、元の位置 x から、距離 d のぶんだけ（dの符号の向きに）歩いた場所。

dがプラスなら右へ、マイナスなら左へ。「−3を足す」とは「左へ3歩く」ことです。下のウィジェットで x と d を別々に動かして、式の項と図の対応を確かめてください（式の x, d, x′ にマウスを乗せると図が光ります）。

WIDGET 2

平行移動 x′ = x + d ― 本編ch2aの「Position」の正体

3. 「マイナスを掛ける」＝向きの反転

次は掛け算。ここも読み替え1つで終わります。「×(−1)」は「向きを反転する」操作です。

  • +3 ×(−1) → −3（右向きだったものが左向きに）
  • −3 ×(−1) → +3（左向きだったものが右向きに。反転の反転で元に戻る）

「マイナス×マイナスがプラスになる」のは規則の暗記ではなく、「反転を2回すると元の向き」という当たり前の話だったのです。×(−2)なら「反転して2倍」。下のウィジェットで倍率スライダーをマイナス側に振って、点が原点の反対側へ跳ぶのを見てください。

x′ = k × x

新しい位置 x′ は、元の位置 x を k 倍した場所。kがマイナスなら原点をはさんで反対側。

WIDGET 3

拡大と反転 x′ = k·x ― 本編ch2aの「Scale」の正体

4. 座標平面の4つの部屋（象限）

XとYの2方向にマイナスがあると、平面は4つの区画に分かれます。学校では第1〜第4象限と呼びました。名前はどうでもよく、大事なのは「符号のペア(+,−)を見れば、どの区画か即答できる」ことだけです。3DCGでは頂点座標が平気でマイナスになるので、この読み取りが無意識にできると楽になります。

下の平面で点をドラッグして、符号ペアの変化と区画の対応を体で覚えてください。

WIDGET 4

4つの区画 ― 符号ペア(±, ±)で場所が決まる

「マイナスの計算は規則を暗記するもの」――暗記した規則は忘れますが、意味は忘れません。「＋は歩く」「×(−1)は反転」。この2つの読み替えだけで、3DCGで出会う負の数の計算はすべて意味から再構成できます。

符号は向き、絶対値は量。足し算は歩き、マイナスの掛け算は反転。
ここまでで一区切りです。難しく感じたら本編第1章に戻ってOK。続けられそうなら、次はいよいよ「回転」を数にする道具、sin / cos へ。
