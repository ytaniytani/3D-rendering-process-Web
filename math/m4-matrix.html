行列 ― ついに「レシピカード」を開ける

本編で13ページかけて「中身は見なくていい」と言い続けたワールド行列・ビュー行列。その中身は、たった数個の数字と、驚くほど単純な規則でした。

  • M3: ベクトル＝数の組を1本の矢印として読む

1. 種明かしの前に: M2の式をもう一度見る

M2で読んだ回転の公式を思い出してください。

x′ = x·cos θ − y·sin θ ／ y′ = x·sin θ + y·cos θ

この2行、よく見るとまったく同じ形をしています。どちらも「元のxに何かを掛けたもの＋元のyに何かを掛けたもの」。違うのは掛ける数（係数）が4つとも別なことだけ。ならば、この4つの係数だけを抜き出して表に並べたら、式を毎回書かなくて済むのでは？――それが行列です。種明かしは、これで終わりです。

[ a b / c d ] 2×2行列（にかけるにぎょうれつ）。変換の係数4つを「上の行=新しいxのレシピ、下の行=新しいyのレシピ」の順に並べた表。a, b, c, dはそれぞれただの数。

2. 行列×ベクトルの規則を「読み下す」

行列をベクトル(x, y)に適用する（掛ける）規則は、次の1つだけです。

x′ = a·x ＋ b·y
y′ = c·x ＋ d·y

新しいxは「aとxの積」と「bとyの積」の和。新しいyは「cとxの積」と「dとyの積」の和。つまり行列の上の行(a, b)は「新しいxの作り方メモ」、下の行(c, d)は「新しいyの作り方メモ」。

本編第2章aで「レシピカード」と呼んでいたものの正体が、この作り方メモの束です。GPUはこのメモを全頂点に一斉適用していたのです（実際の3Dでは4×4=16個の数字になりますが、規則はこの2×2とまったく同じで、行が増えるだけです）。

3. 行列は「空間の変形」そのもの ― 触って掴む

a, b, c, dの4つの数字を変えると何が起きるのか。下のウィジェットは、1×1の正方形（と点）に行列を適用した結果を表示します。まずプリセットボタンを順に押して、4つの数字だけで「拡大」「回転」「せん断（斜め崩し）」「反転」がぜんぶ作れることを見てください。それからスライダーで自由にいじってください。

見るときのコツ: 行列の1列目(a, c)は「横に1進む」の行き先、2列目(b, d)は「縦に1進む」の行き先です。ウィジェットの赤い矢印が1列目、緑の矢印が2列目。正方形はこの2本の矢印に張り付いて変形します。

4. 回転行列 ― M2との合流

プリセットの「回転30°」を押したとき、a, b, c, dには cos30°, −sin30°, sin30°, cos30° が入っていました。つまり回転行列とは、M2の回転の公式の係数をそのまま表に写しただけのものです。

回転行列 = [ cos θ −sin θ / sin θ cos θ ]

上の行（新xのレシピ）が「cos θとーsin θ」、下の行（新yのレシピ）が「sin θとcos θ」。M2の公式の4つの係数を並べただけ。

下のウィジェットではθスライダーだけを回せます。4つの数字が単位円の値（M2）と連動して変わり、正方形が形を崩さずに回る――「回転は、4つの係数の連携プレーだった」ことが見えます。

5. 本編との答え合わせ

これで本編の言葉をすべて数式側から言い直せます。

  • ワールド行列（本編ch2a）＝ Scale・Rotation・Positionの効果を係数に焼き込んだ作り方メモ。頂点全員に同じメモを適用する＝一斉に引っ越し
  • ビュー行列（本編ch2b）＝「世界を逆向きに動かす」変換の係数メモ。カメラの位置と向きから作る
  • 「行列を掛ける」が速い理由 ＝ 中身が掛け算と足し算だけの同じ形の計算で、GPUの「数千人の単純作業」（本編ch3b）に完璧に向いているから

「行列は難解な数学的対象」――3DCGで使う範囲の行列は、「変換の係数を行ごとに並べた表」以上でも以下でもありません。学校で怖かった行列式や逆行列の計算術は、読めるようになる目的には不要です。上の行は新xのレシピ、下の行は新yのレシピ。これだけ持ち帰ってください。

行列＝「新しい各成分の作り方メモ」を行ごとに並べた表。掛けるとは、メモ通りに混ぜること。
